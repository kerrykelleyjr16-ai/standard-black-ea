# Daily Command Center

Run `/daily` to open this. Use it at the start of each day to orient and prioritize.

---

## Daily Startup Sequence

### 1. Orient (5 minutes)
- What are today's top 3 priorities?
- Any deadlines or time-sensitive items?
- Any team check-in today? (Thursday = weekly check-in)
- Any payment due soon? (25th = collect, 28th = due)

### 2. Review Open Items
Check these files for anything outstanding:
- `../context/current-priorities.md` — active focus areas
- `../decisions/log.md` — anything pending from last session
- `10-logs/decision_log.md` — AI-OS level decisions

### 3. Route Today's Tasks
Use `TASK_ROUTER.md` to assign each task:
- Build tasks → `/build`
- Research tasks → `/research`
- Strategy tasks → `/architect`
- Automation tasks → `/automate`

### 4. Execute
Work the list. Log decisions as you go.

### 5. Close (end of day)
- Log any decisions made today
- Note any open items for tomorrow
- Update project READMEs if status changed
