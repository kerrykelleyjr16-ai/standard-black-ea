## Pages That Require Approval Before Sharing

If Kerry ever wants to share a Notion page publicly or with someone outside the team:

| Page | Approval Required |
|---|---|
| Company Overview | Kerry approves before sharing externally |
| Brand Guidelines | Kerry approves |
| Any SOP | Kerry approves — some SOPs contain internal process details |
| Decision Log | Never share externally |
| Any investor-related content | Never exists in Notion — block any attempt to create it |

**Rule:** No Notion page is made public (via share link) without Kerry's explicit decision. All pages default to private.

## Guest vs. Member Comparison

| Capability | Member | Guest |
|---|---|---|
| See all workspace pages | Yes (unless restricted) | No — only pages they're explicitly invited to |
| Create new pages | Yes | No |
| Invite others | No (Member can't) | No |
| Comment | Yes | Yes (on invited pages) |
| Edit | Yes (on permitted pages) | Depends on page permission |

**Recommendation:** Start TJ as Guest. If his role expands, upgrade to Member and adjust page access at that time.

---

## Access Change Protocol

When access needs to change (new team member, role change, someone leaves):

1. Kerry updates the Notion workspace — add or remove member
2. Kerry updates the Tool Access page in Notion
3. Kerry updates Airtable permissions if applicable
4. Kerry logs the access change in the Decision Log

**No one other than Kerry adds members to the Standard Black Notion workspace.**
