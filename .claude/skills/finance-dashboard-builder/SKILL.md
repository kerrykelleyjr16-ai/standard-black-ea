## Workflow

### Step 1 — Identify the Dashboard
Which type does Kerry need? What decisions will it support?

### Step 2 — Collect the Data
Ask Kerry for any numbers not already on file. Don't fabricate — blank cells are fine.

### Step 3 — Build the Dashboard
Output as a clean markdown table with:
- Current period actuals
- Targets where known
- Variance from target (over / under)
- Any red flags highlighted

### Step 4 — Recommend Action
Based on the numbers:
- What's on track?
- What needs attention?
- What's the one thing to fix this week?

## Output Format
Clean markdown dashboard + 3-bullet action summary. No padding.
