# Skill: Weekly Review

**Invoke when:** Kerry runs the 4th Sunday accountability close or any end-of-week review session.

---

## Inputs
- Week being reviewed (dates)
- Any notes from the week Kerry wants to capture

## Workflow

### Step 1 — Read the Week
Read these files:
- `context/current-priorities.md`
- `context/goals.md`
- `decisions/log.md` (entries from this week)
- `projects/team-payment-structure/README.md`
- `ai-os/AUTOMATION_REGISTRY.md`

### Step 2 — Scorecard
For each current priority, report:
- What was the goal this week?
- What actually happened?
- On track / Behind / Blocked?

### Step 3 — Note Business Pulse
- Any tapes reviewed or deals in pipeline?
- Any servicer, seller, or broker conversations?
- Any underwriting runs completed?
- What's the next action in the note mentorship coaching?

### Step 4 — Team Check
- Did the Thursday check-in happen?
- Any blockers for Kody or TJ?
- Team payment status for the month?

### Step 5 — Financial Pulse
- Income from Clay Cooley VW this week (Kerry reports)
- Mentorship credit card balance update (if known)
- Team operating pool — any expenses this week?

### Step 6 — Decisions This Week
Pull any decisions logged this week from `decisions/log.md`. Confirm they're still correct.

### Step 7 — Set Top 3 for Next Week
Kerry names them. Log them. Update `context/current-priorities.md` if anything has shifted.

### Step 8 — Update Context Files
Flag any files that are now stale and need updating. Offer to update them.

## Output Format
Structured report using the WEEKLY_REVIEW.md template. Terse. No fluff. Flag anything off track with specifics.
